APA ITU LOGIKA PROPOSISI?

Konsep Dasar Logika Proposisi

Logika Proposisi mempelajari pernyataan (proposisi) yang dapat bernilai benar (true) atau salah (false).

Definisi:

Proposisi adalah kalimat deklaratif yang memiliki nilai kebenaran, yaitu benar atau salah, tetapi tidak keduanya.

Contoh Proposisi:

"2 + 2 = 4"

Benar

"Bumi berbentuk persegi"

Salah

"Jakarta adalah ibu kota Indonesia"

Benar

Proposisi Majemuk:

Gabungan dari dua atau lebih proposisi sederhana yang dihubungkan dengan operator logika.

Contoh: "Hari ini hujan ∧ saya bawa payung"

P: "Hari ini hujan", Q: "Saya bawa payung" → (P ∧ Q)

OPERATOR DALAM LOGIKA PROPOSISI

Operator logika digunakan untuk menghubungkan proposisi-proposisi menjadi proposisi majemuk.

Konjungsi (∧)

Operator "DAN"

p ∧ q

Disjungsi (∨)

Operator "ATAU"

p ∨ q

Implikasi (→)

Operator "JIKA-MAKA"

p → q

Biimplikasi (↔)

Operator "JIKA DAN HANYA JIKA"

p ↔ q

Negasi (¬)

Operator "TIDAK"

¬p

KONJUNGSI (∧)

Penggabungan dua pernyataan p dan q menggunakan kata hubung logika "DAN"

Notasi: p ∧ q

Dibaca: "p dan q"

Nilai Kebenaran:

p = Benar, q = Benar p ∧ q = Benar
p = Benar, q = Salah p ∧ q = Salah
p = Salah, q = Benar p ∧ q = Salah
p = Salah, q = Salah p ∧ q = Salah

Tabel Kebenaran Konjungsi:

p q p ∧ q
B (Benar) B B
B S (Salah) S
S B S
S S S

Contoh Penerapan dalam Komputer:

Kondisi login: if (username_valid && password_valid) { login(); }

Login hanya berhasil jika username valid DAN password valid.

DISJUNGSI (∨)

Penggabungan dua pernyataan p dan q menggunakan kata hubung logika "ATAU"

Notasi: p ∨ q

Dibaca: "p atau q"

Nilai Kebenaran:

p = Benar, q = Benar p ∨ q = Benar
p = Benar, q = Salah p ∨ q = Benar
p = Salah, q = Benar p ∨ q = Benar
p = Salah, q = Salah p ∨ q = Salah

Tabel Kebenaran Disjungsi:

p q p ∨ q
B B B
B S B
S B B
S S S

Contoh Penerapan dalam Komputer:

Kondisi diskon: if (is_member || total_purchase > 500000) { apply_discount(); }

Diskon diberikan jika pelanggan adalah member ATAU total pembelian lebih dari 500.000.

IMPLIKASI (→)

Kalimat majemuk dengan kata hubung "JIKA ... MAKA ..."

Notasi: p → q

Dibaca: "Jika p maka q"

Nilai Kebenaran:

p = Benar, q = Benar p → q = Benar
p = Benar, q = Salah p → q = Salah
p = Salah, q = Benar p → q = Benar
p = Salah, q = Salah p → q = Benar

Tabel Kebenaran Implikasi:

p q p → q
B B B
B S S
S B B
S S B

Contoh Penerapan dalam Komputer:

Contoh dari PPT:

p: "Program dijalankan"
q: "Output ditampilkan"
Implikasi: if (program_dijalankan) { tampilkan_output(); }

BIIMPLIKASI (↔)

Kalimat majemuk dengan kata hubung "JIKA DAN HANYA JIKA"

Notasi: p ↔ q

Dibaca: "p jika dan hanya jika q"

Nilai Kebenaran:

p = Benar, q = Benar p ↔ q = Benar
p = Benar, q = Salah p ↔ q = Salah
p = Salah, q = Benar p ↔ q = Salah
p = Salah, q = Salah p ↔ q = Benar

Tabel Kebenaran Biimplikasi:

p q p ↔ q
B B B
B S S
S B S
S S B

Contoh Penerapan dalam Komputer:

Validasi input: if ((email_valid && password_valid) ↔ login_allowed) { proceed(); }

Login diperbolehkan jika dan hanya jika email valid dan password valid.

NEGASI (¬)

Kebalikan dari suatu proposisi

Notasi: ¬p atau ~p

Dibaca: "tidak p" atau "bukan p"

Nilai Kebenaran:

p = Benar ¬p = Salah
p = Salah ¬p = Benar

Tabel Kebenaran Negasi:

p ¬p
B S
S B

Contoh Penerapan dalam Komputer:

Validasi input: if (!input_is_empty) { process_input(); }

Proses input jika input tidak kosong (negasi dari "input kosong").

TAUTOLOGI, KONTRADIKSI, DAN KONTINGENSI

Tautologi

Selalu bernilai benar pada semua kondisi

Contoh: p ∨ ¬p

"p atau tidak p" selalu benar

Aplikasi: Validasi algoritma, safety check

Kontradiksi

Selalu bernilai salah pada semua kondisi

Contoh: p ∧ ¬p

"p dan tidak p" selalu salah

Aplikasi: Deteksi error logika

Kontingensi

Kadang benar, kadang salah

Contoh: p → q

Tergantung nilai p dan q

Aplikasi: Kondisi bersyarat dalam program

APLIKASI DALAM KOMPUTER

Pengaplikasian Logika Proposisi dalam Pemrograman

Struktur Kondisi

Penggunaan if-else statement berdasarkan operasi logika

if (is_logged_in && has_permission) {
    // eksekusi kode
}

Validasi Input

Pemeriksaan kebenaran data input pengguna

if (!email.isEmpty() && email.contains("@")) {
    // email valid
}

Query Database

Penggunaan operator logika dalam SQL

SELECT * FROM users 
WHERE status = 'active' 
AND (role = 'admin' OR role = 'supervisor')

Kecerdasan Buatan

Logika proposisi dalam sistem pakar dan inferensi

IF fever AND cough THEN diagnose = 'flu'

Studi Kasus: Platform Kopi Online

Seperti disebutkan dalam PPT: "Usaha kopi butuh platform online yang hemat"

Contoh aturan bisnis:

  • IF (pembelian > 50000 AND member = true) THEN berikan_diskon(10%)
  • IF (stok_kopi > 0 OR stok_alternatif > 0) THEN tampilkan_tersedia
  • IF (pembayaran_valid ↔ order_diproses) THEN update_status
